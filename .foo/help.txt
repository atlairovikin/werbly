werb { "" | abort | help | [un]install | [un]define <formula> | [un]apply <app> | [de]init | show | man <page> | exit | perm-define-all <file> }:

    "":
        (null) Pings werbly.

    abort:
        Undefines `werb`.

    help:
        Show this message.

    [un]install:
        Either installs or uninstalls homebrew into/from an isolated directory.

    [un]define <formula>:
        Either defines or undefines a command from homebrew as <formula>.

    [un]apply <app>:
        Either applies or unapplies an application from homebrew as <app> (as shown with `werb show`).

    [de]init:
        Either initializes or deinitializes the shell environment as nessesary to contain homebrew while using it.

    show:
        Shows contents of the sandbox directory. This can be used to know what to input into `werb [un]apply`.

    man <page>:
        Opens Homebrew's manual file <page> (including any manpages for installed formulae). Depending on the file format, it may render oddly. If <page> argument is not provided, lists all available Homebrew manpages with description using a pager (this is liable to look a bit messy when many formulas are installed).

    exit:
        Undefines werb and unsets any related environmental commands/variables, including `brew`. Note this does not undefine any formulae set using `werb define`.

    perm-define-all <file>:
        Permanently defines brew and all installed formulae by modifing the current user's PATH via (the shell profile) <file>. <file> defualts to `~/.profile` if no argument is provided.
